# 2026-06-29 AIエージェントレビュー方針

## 目的

AI PMとして信頼されるには、AI生成物をそのまま採用するのではなく、レビュー、比較、改善を前提にする必要がある。

## 基本方針

- Codexは一次レビューを実施する。
- Claude、ChatGPTなどの外部AIレビューが可能な場合は結果を比較する。
- 外部AIが利用できない場合は、未実施理由をレビューに残す。
- レビュー差分がある場合は、どの指摘を採用するか明記する。
- AIが生成したIssue、OpenAPI、要件定義は人間承認前提にする。

## レビュー観点

- Product価値
- 技術実現性
- アーキテクチャ
- セキュリティ
- UX
- API整合性
- DB設計
- テスト容易性
- 運用性
- 法務、プライバシー

## 保存対象

- 入力
- プロンプト
- モデル名
- 出力
- レビュー結果
- 採用判断
- 修正履歴
