## 背景

直近のPRタイトル、PR本文、コミットメッセージ、レビュー文書見出しに英語が混在した。AI PMプラットフォームの開発統制では、日本語のIssue、レビュー、改善履歴を前提にしているため、運用文面の言語揺れを放置するとレビュー品質と追跡性が下がる。

## 目的

GitHub Issue、Pull Request、レビューコメント、コミットメッセージ、レビュー文書の主要見出しを日本語で統一する運用ルールを明文化し、直近の英語混在を安全に修正する。

## 完了条件

- AGENTS.mdにGitHub/レビュー/コミット文面の日本語統一ルールが追加されている。
- ISSUE-048で追加したレビュー/セキュリティ文書の主要見出しが日本語化されている。
- PR #53のタイトル/本文が日本語化されている。
- レビュー結果が `docs/review/` へ保存されている。
- main履歴のforce pushが必要な既存コミットメッセージ書き換えは、ユーザー承認なしに実施しない方針が明記されている。

## スコープ

- AGENTS.mdの運用ルール追記
- ISSUE-048関連文書の主要見出し修正
- GitHub PR #53の表示文面修正
- レビュー記録の追加

## 非スコープ

- 過去すべての文書見出し一括翻訳
- main履歴のforce pushによる既存コミットメッセージ書き換え
- 外部仕様名、API名、クラス名、メソッド名、コマンド名の翻訳

## 関連レビュー

- `docs/review/20260706_japanese_artifact_language_governance_review.md`

## レビュー結果

言語統一は見た目の好みではなく、GitHub運用、レビュー監査、Issue台帳検索、非エンジニア参加のしやすさに影響する。日本語を標準とし、技術固有名詞のみ必要に応じて英語を残す方針が妥当である。

## 優先度

P1

理由:

- GitHub上の成果物の言語揺れは、AI PMとしての監査可能性とレビュー継続性を下げる。
- 日本語運用ルールがないと、今後の自動化やサブエージェントレビューでも同じ揺れが再発する。
- force pushによる履歴修正はリスクがあるため、安全な修正範囲を明確にする必要がある。

4. 運用レビューを保存する。
