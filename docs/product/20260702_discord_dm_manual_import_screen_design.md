# Discord DM手動インポート 画面設計

## 目的

ユーザーがDiscord DMを安全に貼り付け、AI整理前に同意・redaction・secret scanを確認し、整理結果をレビューしてからIssue候補や要件候補へ進める画面を設計する。

## 画面原則

- DMは高センシティブ情報として扱う。
- 最初の画面で「貼り付ければ自動で全部進む」と見せない。
- 同意確認、編集、scan、AI整理、レビューを段階的に分ける。
- UI表示は日本語を原則にする。
- 危険操作は通常ボタンではなく、確認状態と理由入力を伴う操作にする。

## 画面構成

### 1. DMインポート作成

入力:

- タイトル
- Discord DMテキスト
- 参加者表示名
- 会話日時
- 同意確認チェック
- 同意文言version表示

主要UI:

- `DMを貼り付け`
- `保存前にスキャン`
- `下書き保存`

必須ブロック:

- 同意確認なしではAI整理へ進めない。
- raw textが空、長すぎる、secret-like検出ありの場合は次ステップを止める。

### 2. スキャン・redaction

表示:

- 検出されたsecret/PII候補
- 行番号または位置ヒント
- severity
- 推奨対応

操作:

- `伏字を適用`
- `編集して再スキャン`
- `リスクを承認して続行` はMVPでは原則非表示。必要な場合はSecurity review後。

空状態:

- `AI整理に進めます`

### 3. AI整理ドラフト

表示:

- 要約
- 決定事項
- 未決事項
- TODO
- Issue候補
- 要件候補
- リスク
- 根拠引用
- confidence

操作:

- `AI整理を生成`
- `編集`
- `レビュー依頼`
- `承認`
- `差し戻し`

重要:

- confidenceが低い項目は自動でIssue候補に送らない。
- source quotesがない決定事項はReview blocker対象にする。

### 4. Review Gate

表示:

- レビューステータス
- 改善点
- 優先順位
- 次アクション
- Issue候補への変換可否

操作:

- `承認してIssue候補へ進む`
- `修正して再レビュー`
- `破棄`

## レスポンシブ設計

- mobileでは貼り付け入力、scan結果、AI整理結果を縦に並べる。
- desktopでは左に原文/編集、右にscan/AI整理結果を置く。
- DM本文とAI整理結果を同時に比較できるが、狭幅では重ならないようにタブ化する。

## アクセシビリティ

- secret/PII検出結果は色だけに依存しない。
- 同意チェックはラベルと説明を明確にする。
- AI整理候補の承認/差し戻しはキーボード操作可能にする。
- エラーは入力欄近くとsummary領域の両方へ出す。

## UXリスク

- 同意確認が形骸化する。
- ユーザーがredactionを面倒に感じて離脱する。
- AI整理結果がもっともらしく見え、誤要約がそのままIssue化される。
- DM相手の個人情報がGitHub Issue候補に残る。

## MVPでやらないこと

- Discordから直接DMを選択するUI
- 添付ファイルpreview
- ブラウザ拡張
- 自動Issue作成
- 相手方に同意依頼を送るワークフロー

## 次に必要な検証

- 1280px/390pxでの視覚QA
- 長文DM貼り付け時の入力領域
- secret/PII検出時のエラー表示
- keyboard操作
- 日本語文言の分かりやすさ
