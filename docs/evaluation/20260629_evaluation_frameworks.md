# 2026-06-29 評価フレームワーク運用

## 基本

評価対象に応じて最適なフレームワークを選ぶ。すべてのレビューで同じ型を使うのではなく、課題に合わせる。

## 適用目安

| 対象 | 推奨フレームワーク |
| --- | --- |
| 事業価値 | Lean Canvas, Business Model Canvas, SWOT, Five Forces |
| 優先順位 | RICE, MoSCoW |
| プロダクト戦略 | G-STACK, HEART |
| アーキテクチャ | C4 Model, ADR, DDD |
| ドメイン設計 | DDD, Event Storming |
| セキュリティ | STRIDE, OWASP Top 10 |
| UX/UI | HEART, WCAG |
| 品質 | ISO25010 |
| 開発生産性 | DORA Metrics, SPACE Framework |

## 禁止事項

- 改善点のないレビュー
- 根拠のない高評価
- Issueに紐づかない改善提案
- レビュー未保存で次工程へ進むこと
