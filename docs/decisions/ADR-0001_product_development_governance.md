# ADR-0001: Issue駆動、API駆動、レビュー必須の開発統制を採用する

## Status

Accepted

## Date

2026-06-29

## Context

本プロダクトは、議事録生成だけでなく、要件定義、Issue生成、API設計、実装、レビュー、リリースまでを自動化するAI PMを目指す。

AI生成物は高速だが、品質、責任、監査、セキュリティの観点で不安定になるリスクがある。

## Decision

以下をプロジェクト標準とする。

- Issueなしで実装しない
- 要件定義の後にOpenAPIを作る
- APIレビュー後にBackend/Frontendへ進む
- 各フェーズ終了時にレビューを保存する
- 技術判断はADRに残す
- GitHub IssueとローカルIssue台帳を紐付ける

## Consequences

良い点:

- AI実装の入力品質が上がる
- レビュー漏れを減らせる
- 意思決定を後から追跡できる
- プロダクト自体の価値仮説と開発プロセスが一致する

悪い点:

- 初速はやや遅くなる
- ドキュメント維持コストが発生する
- レビューの品質が低いと形式主義になる

## Follow-up

- ReviewOps機能をMVPの中核に入れる
- GitHub Issue同期をP0にする
- レビュー未実施ゲートをプロダクト機能として実装する
