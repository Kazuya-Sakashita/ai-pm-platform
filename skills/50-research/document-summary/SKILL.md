# Document Summary

## Purpose

長い文書を、意思決定と次アクションに使える形へ要約する。

## When to use

- docsやIssueコメントが長くなったとき。
- 引き継ぎ、週報、レビューまとめを作るとき。
- 複数資料の差分を整理するとき。

## Inputs

- 対象文書。
- 要約の目的。
- 読者。
- 必要な粒度。

## Process

1. 目的と読者を確認する。
2. 事実、判断、未決、次アクションに分ける。
3. 重要な日付、Issue、PR、CI証跡を残す。
4. 不確かな情報は推測として明記する。
5. 次に読むべき資料を示す。

## Output

- 要約。
- 未決事項。
- 次アクション。
- 参照元一覧。

## Constraints

- secretや不要PIIを含めない。
- 根拠のない断定をしない。
- 長文引用を避ける。

## Checklist

- [ ] 目的に合う粒度。
- [ ] 事実と判断を分けた。
- [ ] Issue/PR/日付が残っている。
- [ ] 次アクションがある。
