# Presentation Planning

## Purpose

AI PM Platformの価値を、相手に合わせたプレゼン構成へ落とし込む。

## When to use

- pitch deck、提案資料、社内説明資料を作る前。
- 章立てやストーリーを整理するとき。
- 図解や比較表を作るとき。

## Inputs

- 想定読者。
- 目的。
- 伝えたい価値。
- 既存docs、競合情報、ロードマップ。

## Process

1. 読者と意思決定を定義する。
2. 1枚目の主張を決める。
3. 課題、解決策、差別化、証拠、次アクションを並べる。
4. 各スライドの目的を1つに絞る。
5. 図解、表、デモ導線を提案する。

## Output

- スライド構成。
- 各スライドの要点。
- 図解案。
- 必要素材リスト。

## Constraints

- 情報を詰め込みすぎない。
- 事実と仮説を分ける。
- 誇張しすぎない。

## Checklist

- [ ] 読者が明確。
- [ ] ストーリーがある。
- [ ] 各スライドの目的がある。
- [ ] 証拠と次アクションがある。
