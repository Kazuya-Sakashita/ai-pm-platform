# Coding Principles

## Purpose

AI PM Platformで安全に継続開発するための基本原則を確認する。

## When to use

- 実装、修正、リファクタ、ドキュメント追加を始める前。
- 複数の進め方があり、作業分割や完了条件に迷うとき。
- `AGENTS.md` の原則を実作業へ落とし込むとき。

## Inputs

- 対象Issue番号と目的。
- 関連する `docs/issue/`、`docs/review/`、ADR。
- 変更対象ファイルと既存実装パターン。

## Process

1. `AGENTS.md` を最優先ルールとして扱う。
2. Issue、レビュー、完了条件を確認する。
3. 既存の設計、命名、責務分離、テスト方針に合わせる。
4. 差分を小さく保ち、関係ないリファクタを避ける。
5. 実装後にテスト、レビュー文書、Issue台帳を更新する。

## Output

- 小さく説明可能な変更差分。
- 検証結果。
- 必要なレビュー文書とIssue更新。

## Constraints

- Issueなしで実装しない。
- レビューなしで次工程へ進まない。
- 日本語運用ルールを守る。
- 既存のユーザー変更を勝手に戻さない。

## Checklist

- [ ] 対象Issueがある。
- [ ] 既存パターンを確認した。
- [ ] 変更範囲が過剰でない。
- [ ] テストまたは検証を実施した。
- [ ] docsとIssue台帳を更新した。
