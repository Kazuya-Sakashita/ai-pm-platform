# G-STACK Analysis

## Purpose

レビューや意思決定をG-STACKで短く構造化する。

## When to use

- フェーズレビュー。
- 設計判断。
- Issue優先順位。
- リリース判定。

## Inputs

- 対象Issue。
- 対象成果物。
- 判断したい問い。

## Process

1. Goalを1文で書く。
2. Strategyを選ぶ。
3. Tacticsを具体手順にする。
4. Assessmentで良い点と問題点を評価する。
5. Conclusionで判定する。
6. Knowledgeで次回に残す学びを書く。

## Output

- G-STACKレビュー。
- 判定。
- 次アクション。

## Constraints

- 甘い評価で終わらない。
- 改善点と改善案を必ず含める。
- Issue番号を記録する。

## Checklist

- [ ] G/S/T/A/C/Kが揃った。
- [ ] 改善点がある。
- [ ] 次アクションがある。
- [ ] Issue番号がある。
